Use this checklist for the current refinement pass.

- [ ] Public offer architecture matches the current package ladder.
- [ ] Broader platform and systems capability is secondary to the current website-package entry offer.
- [ ] Home, Website Packages, About, Start an Engagement, and Selected Engagements avoid systems-heavy lead framing.
- [ ] No unsupported prices appear in UI, metadata, schema, alt text, helper copy, or selectors.
- [ ] No unsupported proof metrics, dollar outcomes, durations, or named-client claims appear.
- [ ] No proprietary prompts, scoring logic, internal methods, or review secrets are exposed.
- [ ] Privacy and terms language stay within verified implementation and legal boundaries.
- [ ] Careers posture remains selective and does not imply indiscriminate hiring.
- [ ] Public copy uses buyer-safe language and explains or removes specialist shorthand.
- [ ] Release notes and evidence artifacts do not overstate approval or readiness.
