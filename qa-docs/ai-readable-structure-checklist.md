# AI-Readable Structure Checklist

Use this checklist for the current refinement pass.

- [ ] Each public route has one `h1`.
- [ ] No heading hierarchy skips exist within a route.
- [ ] Shell uses semantic landmarks: header, nav, main, footer.
- [ ] Page partials use semantic sectioning and article structure where appropriate.
- [ ] Canonical URL resolves from runtime config when configured.
- [ ] `Organization` JSON-LD is present and valid.
- [ ] `WebSite` JSON-LD is present and valid.
- [ ] Route-level page JSON-LD is present and valid.
- [ ] Breadcrumb JSON-LD is present on non-home routes if supported by the shell.
- [ ] Public metadata and schema use `Website Packages` instead of `Capabilities` where the label changed.
- [ ] Home, Website Packages, About, and Start an Engagement remain readable when extracted as plain text.
- [ ] Acronyms and specialist shorthand are removed, replaced, or explained in buyer-safe language.
- [ ] Structured data does not leak unapproved prices or unsupported proof.
- [ ] Hreflang strategy is reviewed before removing `noindex`.
