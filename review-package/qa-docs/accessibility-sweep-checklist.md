# Accessibility Sweep Checklist

Use this checklist for the current refinement pass.

- [ ] Route changes move focus to the primary heading.
- [ ] Active nav links expose `aria-current="page"`.
- [ ] Mobile nav keeps focus contained while open.
- [ ] Mobile nav returns focus to the toggle when closed with Escape.
- [ ] Skip link is visible on focus and lands on main content.
- [ ] Website Packages heading hierarchy is logical and does not skip levels.
- [ ] Start an Engagement form labels, legends, helper text, and errors remain associated correctly.
- [ ] Inquiries form labels and errors remain associated correctly.
- [ ] Workflow-specific privacy/status notices are announced appropriately.
- [ ] Success and mode states use `aria-live` supportably.
- [ ] Touch targets remain usable on mobile, especially nav controls and form actions.
- [ ] Careers does not expose an inaccessible or unsupported applicant form.
- [ ] Keyboard-only navigation works through home, Website Packages, about, Start an Engagement, inquiries, and careers.
- [ ] `prefers-reduced-motion` disables non-essential transitions and page animation.
- [ ] Screen-reader review is scheduled or completed for navigation and forms before release.
